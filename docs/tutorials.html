Getting Started

This tutorial will walk you through setting up the Penetration Test Suite and performing your first scan.

Step 1: Installation

# Clone the repository
git clone https://github.com/samueldk12/penetration-test.git
cd penetration-test

# Install Python dependencies
pip install -r requirements.txt

# Verify installation
python3 penetration-test.py --version

Step 2: Initialize Configuration

# Create default config file
python3 penetration-test.py config --init

# Review configuration
python3 penetration-test.py config --show

Step 3: Check Available Plugins

# List all plugins
python3 penetration-test.py plugins

# List with descriptions
python3 penetration-test.py plugins -v
✓ Success! You're now ready to start scanning.

Your First Scan

Simple Target Scan

Let's perform a basic scan on a target domain:

# Basic scan with all plugins
python3 penetration-test.py scan example.com --all-plugins

# Output will show:
# - Plugin execution progress
# - Findings as they're discovered
# - Summary statistics

Scanning Specific Categories

You can limit the scan to specific categories:

# Only reconnaissance plugins
python3 penetration-test.py scan example.com --categories recon

# Only vulnerability scanning
python3 penetration-test.py scan example.com --categories vuln_scan

# Multiple categories
python3 penetration-test.py scan example.com --categories recon,vuln_scan

Running Specific Plugins

# Run single plugin
python3 penetration-test.py scan example.com --plugin nuclei_scanner

# Run multiple plugins
python3 penetration-test.py scan example.com --plugin nuclei_scanner,nmap_scanner
ℹ️ Tip: Use --verbose flag for detailed output: python3 penetration-test.py scan example.com --all-plugins --verbose

Advanced Scanning Techniques

Complete Scan with All Options

# Comprehensive scan
python3 penetration-test.py scan example.com \
    --all-plugins \
    --complete \
    --verbose \
    --threads 10 \
    --timeout 600

Scanning Multiple Targets

Create a bash script to scan multiple targets:

#!/bin/bash
# scan-multiple.sh

TARGETS=(
    "example.com"
    "test.com"
    "demo.com"
)

for target in "${TARGETS[@]}"; do
    echo "Scanning $target..."
    python3 penetration-test.py scan "$target" --all-plugins
    echo "---"
done

Custom Plugin Arguments

Some plugins accept custom arguments:

# Pass custom arguments via config
python3 penetration-test.py scan example.com \
    --plugin nmap_scanner \
    --config custom-config.yaml

Excluding Plugins

# Run all except specific plugins
python3 penetration-test.py scan example.com \
    --all-plugins \
    --exclude nikto_scanner,nmap_scanner

OSINT Investigation

Basic OSINT Gathering

# Standard OSINT investigation
python3 penetration-test.py osint example.com

# Deep investigation (more comprehensive)
python3 penetration-test.py osint example.com --deep

What OSINT Collects

  • Domain Information: WHOIS records, DNS records, SSL certificates
  • Subdomains: Active subdomain enumeration
  • Email Addresses: Harvested from public sources
  • Social Media: Company presence and employee profiles
  • Technologies: Web technologies and frameworks used
  • Leaked Data: Check for breaches and exposed data

Combining OSINT with Scanning

# First, gather OSINT
python3 penetration-test.py osint example.com --deep

# Then, scan discovered subdomains
# (Subdomains are saved in database)

# Run full scan
python3 penetration-test.py scan example.com --all-plugins

Using the Web Interface

Starting the Web Server

# Default (localhost:5000)
python3 penetration-test.py web

# Custom port
python3 penetration-test.py web --port 8080

# Allow external access
python3 penetration-test.py web --host 0.0.0.0 --port 8080

# Debug mode
python3 penetration-test.py web --debug

Creating Your First Scan

  1. Open browser at http://localhost:5000
  2. Click "New Scan" button
  3. Enter target domain
  4. Select scan type (All Plugins / By Category / Specific Plugins)
  5. Click "Start Scan"
  6. Monitor progress in real-time

Managing Multiple Targets

The web interface supports multi-tab management:

  • Click "+ New Target" to add a tab
  • Each tab maintains independent configuration
  • Run scans in parallel across tabs
  • Switch between targets easily

Adding Notes

1. Navigate to "Notes" section
2. Click "Add Note"
3. Select target
4. Write your findings/observations
5. Click "Save Note"

Notes are searchable and exportable!

Creating Custom Plugins

Python Plugin Example

# tools/plugins/my_scanner.py
from plugin_base import PluginBase

class MyScanner(PluginBase):
    def __init__(self):
        super().__init__(
            name="my_scanner",
            description="My custom security scanner",
            category="vuln_scan",
            version="1.0.0"
        )

    def run(self, target, **kwargs):
        """Scan implementation"""
        self.logger.info(f"Scanning {target}...")

        findings = []

        # Your scanning logic here
        # ...

        return {
            'success': True,
            'findings': findings,
            'target': target
        }

scan(target).then(result => {
    console.log(JSON.stringify(result));
});

Go Plugin Example

// tools/go_plugins/my_scanner.go
// @name my_scanner
// @description My custom Go scanner
// @category recon
// @version 1.0.0

package main

import (
    "encoding/json"
    "fmt"
    "os"
)

type Result struct {
    Success bool     `json:"success"`
    Data    []string `json:"data"`
}

func main() {
    if len(os.Args) > 1 && os.Args[1] == "--info" {
        fmt.Println(`{"name":"my_scanner","version":"1.0.0"}`)
        return
    }

    // Scanning logic
    result := Result{
        Success: true,
        Data:    []string{},
    }

    json.NewEncoder(os.Stdout).Encode(result)
}

Installing Your Plugin

# Place plugin in appropriate directory
cp my_scanner.py tools/plugins/

# Verify it's detected
python3 penetration-test.py plugins

# Test it
python3 penetration-test.py scan example.com --plugin my_scanner

Setting Up Notifications

Telegram Notifications

  1. Create a bot with @BotFather
  2. Get your bot token
  3. Get your chat_id from @userinfobot
  4. Update config.yaml:
notifications:
  enabled: true
  telegram:
    enabled: true
    bot_token: "YOUR_BOT_TOKEN"
    chat_id: "YOUR_CHAT_ID"

Email Notifications

notifications:
  email:
    enabled: true
    smtp_host: smtp.gmail.com
    smtp_port: 587
    smtp_user: your@email.com
    smtp_password: your_app_password
    from_email: your@email.com
    to_emails:
      - recipient@email.com
⚠️ Gmail Users: Use App Passwords instead of your regular password. Enable 2FA and generate an app password in your Google Account settings.

WhatsApp Notifications (CallMeBot)

  1. Add CallMeBot to your contacts: +34 644 28 77 64
  2. Send message: "I allow callmebot to send me messages"
  3. Save your API key
  4. Update config:
notifications:
  whatsapp:
    enabled: true
    api_key: "YOUR_API_KEY"
    phone: "+1234567890"

Generating Reports

Basic Report Generation

# Generate comprehensive report
python3 penetration-test.py report --type comprehensive

# Specific formats
python3 penetration-test.py report --format json
python3 penetration-test.py report --format html
python3 penetration-test.py report --format markdown

# Multiple formats
python3 penetration-test.py report --format json html markdown

Filtering Reports

# By severity
python3 penetration-test.py report --severity critical high

# By domain
python3 penetration-test.py report --domain example.com

# By vulnerability type
python3 penetration-test.py report --vuln-type xss sqli

# By date range
python3 penetration-test.py report \
    --from 2024-01-01 \
    --to 2024-12-31

Custom Report Output

# Specify output directory
python3 penetration-test.py report \
    --output /path/to/reports/ \
    --format html

# Include raw data
python3 penetration-test.py report \
    --type comprehensive \
    --include-raw

Automation & CI/CD

Cron Job Setup

# Edit crontab
crontab -e

# Add daily scan at 2 AM
0 2 * * * cd /path/to/penetration-test && python3 penetration-test.py scan example.com --all-plugins

# Weekly comprehensive scan
0 3 * * 0 cd /path/to/penetration-test && python3 penetration-test.py scan example.com --all-plugins --complete

GitHub Actions Integration

# .github/workflows/security-scan.yml
name: Security Scan

on:
  schedule:
    - cron: '0 2 * * *'  # Daily at 2 AM
  workflow_dispatch:

jobs:
  scan:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3

      - name: Set up Python
        uses: actions/setup-python@v4
        with:
          python-version: '3.10'

      - name: Install dependencies
        run: pip install -r requirements.txt

      - name: Run scan
        run: |
          python3 penetration-test.py scan ${{ secrets.TARGET_DOMAIN }} --all-plugins

      - name: Generate report
        run: |
          python3 penetration-test.py report --format html

      - name: Upload report
        uses: actions/upload-artifact@v3
        with:
          name: security-report
          path: reports/

Docker Integration

# Create Dockerfile
FROM python:3.10-slim

WORKDIR /app
COPY . /app

RUN pip install -r requirements.txt

ENTRYPOINT ["python3", "penetration-test.py"]

# Build image
docker build -t pentest-suite .

# Run scan
docker run --rm pentest-suite scan example.com --all-plugins
