Find answers to common questions about the Penetration Test Suite.

General Questions

What is the Penetration Test Suite?

The Penetration Test Suite is a comprehensive security testing framework that integrates multiple reconnaissance, vulnerability scanning, and OSINT tools into a unified platform. It supports plugins in Python, JavaScript, and Go, providing flexibility and extensibility for security professionals.

Who should use this tool?

This tool is designed for:

  • Penetration testers and ethical hackers
  • Security researchers and analysts
  • Bug bounty hunters
  • Red team operators
  • Security students and learners
  • SOC teams for reconnaissance

Important: Always ensure you have proper authorization before testing any systems.

Is this tool free and open source?

Yes! The Penetration Test Suite is completely free and open source, released under the MIT License. You can use, modify, and distribute it freely. Contributions are welcome on our GitHub repository.

What makes this tool different from others?

Key differentiators include:

  • Multi-language plugins: Python, JavaScript, and Go support
  • Unified interface: Both CLI and web interface available
  • Extensible architecture: Easy plugin installation from GitHub
  • Notification system: Telegram, Email, WhatsApp, Webhook support
  • Comprehensive reporting: Multiple formats (JSON, HTML, Markdown, PDF)
  • Active development: Regular updates and community support

Installation

What are the system requirements?

Minimum Requirements:

  • Python 3.8 or higher
  • 2GB RAM (4GB recommended)
  • 1GB free disk space
  • Linux, macOS, or Windows with WSL
  • Internet connection for plugin downloads

Optional:

  • Node.js 14+ for JavaScript plugins
  • Go 1.16+ for Go plugins
  • Docker for containerized deployment
How do I install the tool? 
# Clone the repository
git clone https://github.com/samueldk12/penetration-test.git
cd penetration-test

# Install Python dependencies
pip install -r requirements.txt

# Verify installation
python3 penetration-test.py --help
Can I install this on Windows?

Yes, but we recommend using Windows Subsystem for Linux (WSL) for the best experience. Some tools (like Nmap, Nuclei) may require native Linux binaries. 

# Install WSL2
wsl --install

# Inside WSL, follow the Linux installation steps
git clone https://github.com/samueldk12/penetration-test.git
cd penetration-test
pip install -r requirements.txt
Do I need to install external tools manually?

Most tools are installed automatically as plugins. However, some core tools should be installed system-wide:

  • Nmap: sudo apt install nmap
  • Nuclei: Installed via plugin system
  • FFUF: Installed via plugin system
  • Katana: Installed via plugin system

The tool will warn you if a required dependency is missing.

Usage

How do I perform a basic scan? 
# Basic scan with all plugins
python3 penetration-test.py scan example.com --all-plugins

# Scan specific category
python3 penetration-test.py scan example.com --categories recon

# Scan with specific plugins
python3 penetration-test.py scan example.com --plugins subdominator,nmap_scanner
How do I view scan results?

Results are stored in the results/ directory by default. You can view them using:

# List all scans
python3 penetration-test.py list

# View specific scan
python3 penetration-test.py view <scan_id>

# Generate report
python3 penetration-test.py report --scan <scan_id> --format html
Can I scan multiple targets at once?

Yes! You can provide multiple targets or use a file:

# Multiple targets
python3 penetration-test.py scan example.com,target2.com,target3.com --all-plugins

# From file
echo "example.com" > targets.txt
echo "target2.com" >> targets.txt
python3 penetration-test.py scan --file targets.txt --all-plugins
How do I schedule automated scans?

Use cron jobs for scheduling:

# Edit crontab
crontab -e

# Run daily at 2 AM
0 2 * * * cd /path/to/penetration-test && python3 penetration-test.py scan example.com --all-plugins --notify telegram

# Run weekly on Sunday
0 0 * * 0 cd /path/to/penetration-test && python3 penetration-test.py scan example.com --all-plugins --report

Plugins

How do I install a plugin? 
# Install from GitHub
python3 penetration-test.py plugin install https://github.com/user/plugin-repo

# Install specific type
python3 penetration-test.py plugin install https://github.com/user/plugin-repo --type python

# Force overwrite existing
python3 penetration-test.py plugin install https://github.com/user/plugin-repo --force
How do I create a custom plugin?

Create a plugin manifest file (plugin.json):

{
  "name": "my_plugin",
  "version": "1.0.0",
  "description": "My custom plugin",
  "type": "python",
  "category": "recon",
  "author": "Your Name",
  "entrypoint": "plugin.py",
  "dependencies": []
}

See the tutorial section for complete examples.

Which programming languages can I use for plugins?

The suite supports three languages:

  • Python: Most flexible, best for integrations and API calls
  • JavaScript: Great for web-based testing and Node.js tools
  • Go: High performance, ideal for network tools and scanners
How do I update plugins? 
# Update all plugins
python3 penetration-test.py plugin update --all

# Update specific plugin
python3 penetration-test.py plugin update my_plugin

# List plugins with available updates
python3 penetration-test.py plugin list --outdated

Web Interface

How do I access the web interface? 
# Start web interface
python3 penetration-test.py web

# Custom host and port
python3 penetration-test.py web --host 0.0.0.0 --port 8080

# Enable debug mode
python3 penetration-test.py web --debug

Then open your browser to http://localhost:5000

Is the web interface secure?

The web interface is designed for local use by default. For production deployments:

  • Use HTTPS with a reverse proxy (nginx, Apache)
  • Implement authentication (basic auth, OAuth)
  • Restrict access by IP address
  • Run behind a VPN
  • Use firewall rules to limit access
Can I access the web interface remotely?

Yes, but with security precautions:

# Allow remote access (use with caution!)
python3 penetration-test.py web --host 0.0.0.0 --port 5000

# Better: Use SSH tunnel
ssh -L 5000:localhost:5000 user@remote-server
# Then access http://localhost:5000 on your local machine

Troubleshooting

Plugin execution failed - what should I check?

Common issues and solutions:

  • Missing dependencies: Check plugin.json and install required packages
  • Permission issues: Ensure plugin files are executable
  • Path problems: Verify the plugin is in the correct directory
  • Syntax errors: Check plugin code for errors
# Debug mode for detailed output
python3 penetration-test.py scan example.com --plugin my_plugin --debug

# Check plugin status
python3 penetration-test.py plugin list --verbose
Scans are running slowly - how to improve performance?

Performance optimization tips:

  • Use specific plugins instead of --all-plugins
  • Increase thread count: --threads 20
  • Reduce timeout values: --timeout 5
  • Skip unnecessary plugins
  • Run scans during off-peak hours
  • Use faster DNS resolvers
API rate limiting issues - what to do?

Handle rate limits:

  • Add delays between requests: --delay 2
  • Use API keys when available
  • Distribute scans across multiple API keys
  • Run scans over longer time periods
  • Use proxy rotation
How do I enable verbose logging? 
# Enable debug output
python3 penetration-test.py scan example.com --debug --verbose

# Log to file
python3 penetration-test.py scan example.com --log-file scan.log

# Different log levels
python3 penetration-test.py scan example.com --log-level DEBUG

Legal & Ethics

Is it legal to use this tool?

Yes, when used legally and ethically. However:

⚠️ CRITICAL: You MUST have explicit written authorization before testing any systems you don't own. Unauthorized access is illegal in most jurisdictions.

Legal use cases include:

  • Testing your own systems
  • Authorized penetration testing engagements
  • Bug bounty programs (within scope)
  • Security research with permission
  • Educational purposes on your own lab
What authorization do I need?

Required documentation:

  • Written contract or letter of authorization
  • Clearly defined scope (IP ranges, domains)
  • Testing time windows
  • Emergency contact information
  • Rules of engagement
  • Data handling agreement
Can I use this for bug bounty programs?

Yes! This tool is excellent for bug bounty hunting. However:

  • Always read and follow the program's scope and rules
  • Stay within authorized targets only
  • Respect rate limits and don't cause DoS
  • Don't access or modify user data
  • Report findings responsibly
What if I accidentally scan something unauthorized?

Stop immediately and:

  • Cease all scanning activities
  • Document what happened
  • Notify the system owner if possible
  • Delete any collected data
  • Learn from the mistake and double-check scope going forward

Advanced Topics

Can I integrate this with CI/CD pipelines?

Yes! Example GitHub Actions workflow:

name: Security Scan
on: [push]
jobs:
  scan:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - name: Setup Python
        uses: actions/setup-python@v2
        with:
          python-version: '3.9'
      - name: Install dependencies
        run: pip install -r requirements.txt
      - name: Run scan
        run: python3 penetration-test.py scan ${{ secrets.TARGET }} --plugins nuclei_scanner
      - name: Upload results
        uses: actions/upload-artifact@v2
        with:
          name: scan-results
          path: results/
How do I use proxies and VPNs? 
# HTTP proxy
python3 penetration-test.py scan example.com --proxy http://proxy:8080

# SOCKS5 proxy
python3 penetration-test.py scan example.com --proxy socks5://proxy:1080

# Proxy with authentication
python3 penetration-test.py scan example.com --proxy http://user:pass@proxy:8080

# Environment variable
export HTTP_PROXY=http://proxy:8080
export HTTPS_PROXY=http://proxy:8080
Can I run this in Docker?

Yes! Create a Dockerfile:

FROM python:3.9-slim

RUN apt-get update && apt-get install -y \
    nmap \
    git \
    && rm -rf /var/lib/apt/lists/*

WORKDIR /app
COPY requirements.txt .
RUN pip install -r requirements.txt

COPY . .

ENTRYPOINT ["python3", "penetration-test.py"]
# Build and run
docker build -t pentest-suite .
docker run -v $(pwd)/results:/app/results pentest-suite scan example.com --all-plugins
